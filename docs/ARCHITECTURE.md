# 架构简述

> 本文档为初赛 DEMO 提交材料的补充说明，仅描述系统架构与数据流，不含源码。

---

## 🎯 核心定位

把电力施工单位"工程师每周手敲几十页 Word 文档"这件重复劳动，压缩为「选工程 → 点按钮 → 5 秒下载标准文档」。

## 🧱 系统分层

```
┌─────────────────────────────────────────────────────┐
│ 前端 (Vue 3 + Element Plus)                        │
│ - 5 个文档生成模块页面                              │
│ - 工程 / 人员 / 工艺基础数据管理                     │
│ - 文档预览 (mammoth → HTML)                        │
└────────────────┬────────────────────────────────────┘
                 │ REST API
┌────────────────▼────────────────────────────────────┐
│ 后端 (FastAPI + SQLAlchemy)                        │
│ ┌──────────────┐ ┌──────────────┐ ┌──────────────┐ │
│ │ data_service │→│doc_generator │→│ output/      │ │
│ │  数据组合     │ │  Word/XLSX    │ │  生成文件     │ │
│ └──────────────┘ └──────────────┘ └──────────────┘ │
└────────────────┬────────────────────────────────────┘
                 │ ORM
┌────────────────▼────────────────────────────────────┐
│ 数据层 (SQLite)                                     │
│ - 工程 / 人员 / 工艺 / 设备 / 文档 5 类表          │
│ - 共享同一份真源，5 份文档自动引用                   │
└─────────────────────────────────────────────────────┘
```

## 🔄 数据流

1. **前端**：用户选择工程 → 触发 `POST /api/{module}/generate`
2. **后端**：`data_service` 从 DB 拉取工程、人员、工艺、设备等真源数据，自动组合成占位符字典
3. **后端**：`doc_generator` 把字典套入 Word/XLSX 模板 → 输出文件落到 `backend/output/`
4. **前端**：返回 `{id, filename, download_url}` → 用户点下载

## 📋 5 个文档生成模块

| 模块 | 关键占位符（示例） |
|---|---|
| 施工组织设计 | 项目名称、5 个角色（工程负责人/技术员/安全员/材料员/应急组长）、12 项工艺内容 |
| 项目勘察单 | 勘察负责人、勘察人员、编号、勘察日期、危险源、安全措施 |
| 技术交底 | 项目名、班组、主持人、交底日期、交底内容 |
| 安全交底 | 项目名、班组、安全员、交底日期、危险源、防控措施 |
| 施工进度横道图 | 任务名、起止日期、进度条、关键路径（XLSX 输出） |

## 🔑 关键设计决策

### 1. 一份真源 → N 份文档
所有文档共享同一份 SQLite 真源。新增一次人员/工艺，5 份文档自动引用 — 杜绝交叉引用错漏。

### 2. 模板与代码解耦
- `templates/*_v2.docx` 是带 `{{placeholder}}` 的 Word 模板
- 代码只负责"查找占位符 → 替换为字典值"
- 新增文档类型只需新增模板，**不动业务代码**

### 3. 自动拉取代替手动选择
- 用户在前端**只选工程**，不选人员/工艺
- 后端按 `ProjectMember → Worker → 全局库` 三级兜底自动匹配角色
- 体验上"一键生成"，而不是"打勾 12 项"

### 4. 标准协议
- API 列表型接口统一返回 `{items, total, page, page_size}`
- 文档主键字段 `id`（不是 `document_id`）
- 端口固定 `8080`（已规避 `8000` 冲突）

## 🛠️ 技术选型理由

| 维度 | 选择 | 原因 |
|---|---|---|
| 前端框架 | Vue 3 | 学习曲线平缓，Element Plus 组件齐全 |
| 构建工具 | Vite | 秒级热重载，5 分钟出构建产物 |
| 后端框架 | FastAPI | 自动生成 OpenAPI 文档，类型提示完整 |
| 数据库 | SQLite | 零部署，单文件可随项目分发 |
| 文档库 | python-docx | 业界标准，支持模板替换 |
| 预览 | mammoth | DOCX → HTML 实时预览，免去 PDF 中转 |
